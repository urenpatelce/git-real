Ui

This project was generated with Angular CLI version 7.3.0.

Pre-requisites

Run npm config set @bmc-ux:registry http://cdn.bmc.com:4873 to set the BMC adapt repository(One time setup). Run npm ci to download dependencies specified in package-lock.json. Need to run this whenever there is a change in package.json and package-lock.json.

Development server

From current directory, Run ng build common to build the common library first. Need to build common library whenever there is a change in common. Run ng serve project-name for a dev server. Navigate to http://localhost:4200/. The app will automatically reload if you change any of the source files. For eg., running bcan project, run ng serve bcan.

Code scaffolding

Change directory to 'projects/project-name/src/app'. Run ng generate component component-name --skip-tests --style=scss to generate a new component without tests file and with scss styling. You can also use ng generate directive|pipe|service|class|guard|interface|enum|module. For eg., creating component test in admin directory of bcan, run cd projects/bcan/src/app; ng generate component admin/test --skip-test --style=scss. Run ng generate component component-name --dry-run --skip-tests --skip-import --style=scss to dry run your command and check which files will be created on actual generation without actually creating the files.

Build

The build artifacts will be stored in the dist/ directory. Use the --prod flag for a production build. Run ng build common to build the common library. Run ng build project-name --prod, for production build of project. For eg., for production build of bcan, ng build bcan --prod.

NPM scripts

These are shorthands for running the commands specified above. For building common, run npm run release-common. For building any project, npm run build project-name. For starting dev server for a project, npm run start project-name. (Common must be built first). For production build of project along with common, npm run release-project-name. eg., npm run release-bcan. For running lint on overall UI folder, npm run lint. For running lint on specific project, npm run lint project-name. eg., npm run lint bcan.

Development environment

Rest API's are deployed on local server first, i.e Run BNA in cygwin. Build and serve the project in cmd. To access the UI, goto http://localhost:4200/ from a security disabled web browser to allow cross oirigin access for rest api's. (ISSUE ONLY FOR DEV). Get the browser certificate on the 'http://localhost:8443' restapi page first to allow access of the rest api's in the same security disabled web browser. You are now set to use the new UI with the Rest api's.
